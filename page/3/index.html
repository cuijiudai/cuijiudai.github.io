Sass和Scss的关系

Sass 有两种语法规则(syntaxes),目前新的语法规则（从 Sass 3开始）被称为 “SCSS”( 时髦的css（Sassy CSS）),它是css3语法的的拓展级，就是说每一个语法正确的CSS3文件也是合法的SCSS文件，SCSS文件使用.scss作为拓展名。
第二种语法别成为缩进语法（或者 Sass），它受到了 Haml的简洁精炼的启发，它是为了人们可以用和css相近的但是更精简的方式来书写css而诞生的。它没有括号，分号，它使用 行缩进 的方式来指定 css 块，虽然sass不是最原始的语法，但是缩进语法将继续被支持，在缩进语法的文件以 .sass 为拓展名。

安装Ruby 和 Sass

首先，让我们获得Sass并搭配环境.如果你使用 OS X,那么你已经安装了Ruby. Windows 用户可以通过Windows安装程序来安装, Linux可以通过包管理(apt-get)来安装.

当Ruby安装好后,你可以安装 Sass 了，运行：

    <!--more-->

gem install haml`</pre>

#### 你的第一个Sass样式文件

我们可以开始了，先创建一个简单的SCSS 文件.因为 SCSS是CSS3 的一个拓展, 我们的第一个文件将从普通的 CSS开始.在你的[编辑器](http://sass-lang.com/editors.html)中创建一个名为**`style.scss`** 的文件,添加下面的代码:

        <pre class="block sass">/* style.scss */
#navbar {
  width: 80%;
  height: 23px;
}</pre>

如何将这个Sass文件转换为CSS 文件？,运行：

        <pre>`sass --watch style.scss:style.css`</pre>

现在无论何时你修改`style.scss`, Sass 都会自动的更新 `style.css` 文件的改变.以后当你有多个Sass文件的时候，你可以监听整个文件夹:

        <pre>`sass --watch stylesheets/sass:stylesheets/compiled

特点

虽然每一个 css 文件都是合法的Sass 文件是一件不错的事, 但这显然不是你学习SCSS 的原因.你学习它是因为它提供的比CSS 更多. SCSS 在CSS 的基础上提供了所有各种有用的特性. 这个教程将带领你学习一些最基础的, 另外还提供一些示例，你可以将它添加到 style.scss 并动手测试一下.

嵌套

当你书写 CSS时,你会经常遇到一些选择器有相同的开头. 比如，你或许要写 “#navbar ul”, “#navbar li”, and “#navbar li a”的样式. 重复一遍遍书写开头的代码很是无聊啊, 特别是它还很长的时候.

Sass 可以让你避免这些问题，他是通过将子选择器嵌套到父选择器实现的.

<div class="compare">
  <pre class="sass">/* style.scss */

#navbar {
width: 80%;
height: 23px;

ul { list-style-type: none; }
li {
float: left;
a { font-weight: bold; }
}
}

/ style.css /

#navbar {
width: 80%;
height: 23px; }

#navbar ul {
list-style-type: none; }

#navbar li {
float: left; }

#navbar li a {
  font-weight: bold; }


注意输出是如何格式化来反映原来的选择器嵌套的.这是默认的CSS样式,对所有种类的 CSS 格式化可以查看其他样式参考甚至有一个是专门用来压缩css的！！

你也可以嵌套属性,
所以你就不用一直重复书写像 “border-left-“ 一样的东东了.

<div class="compare">
  <pre class="sass">/* style.scss */

.fakeshadow {
border: {
style: solid;
left: {
width: 4px;
color: #888;
}
right: {
width: 2px;
color: #ccc;
}
}

/ style.css /

.fakeshadow {
border-style: solid;
border-left-width: 4px;
border-left-color: #888;
border-right-width: 2px;
border-right-color: #ccc; }

父选择器引用

伪类如何,像:hover? 它和它得到父选择器之间没有空格, 但仍然可以嵌套他们.你使用Sass 的特殊字符&amp;就可以了.在一个选择器里, &amp;将会逐字被父选择器替换

<div class="compare">
  <pre class="sass">/* style.scss */

a {
color: #ce4dd6;
&:hover { color: #ffb3ff; }
&:visited { color: #c458cb; }
}

/ style.css /

a {
color: #ce4dd6; }
a:hover {
color: #ffb3ff; }
a:visited {
color: #c458cb; }

变量

Sass允许你定义整个样式表可用的变量. 变量以 $开始，变量声明像属性似的.他们可以被赋值任何css允许使用的属性值 , 像颜色,数字(带有 单位),或文本.

<div class="compare">
  <pre class="sass">/* style.scss */

$main-color: #ce4dd6;
$style: solid;

#navbar {
border-bottom: {
color: $main-color;
style: $style;
}
}

a {
color: $main-color;
&:hover { border-bottom: $style 1px; }
}

/ style.css /

#navbar {
border-bottom-color: #ce4dd6;
border-bottom-style: solid; }

a {
color: #ce4dd6; }
a:hover {
border-bottom: solid 1px; }

变量可以使你重复使用 颜色，大小,和其他值. 这意味着改变将是最小的.如：改变颜色或者大小可以一次完成，而不是在整个样式表中多次更改

Operations and Functions

除了仅仅使用定义的变量值, 你还可以使用预定义函数更改和一起使用它们.这允许你动态的计算元素的尺寸或者颜色.

数字可以使用标准的数学符号 (+, -, *, /, and %) , 即使他们有单位. 对于颜色, 有各种有用的函数来改变 亮度, 色调, 饱和度, 和 更多.

<div class="compare">
  <pre class="sass">/* style.scss */

#navbar {
$navbar-width: 800px;
$items: 5;
$navbar-color: #ce4dd6;

width: $navbar-width;
border-bottom: 2px solid $navbar-color;

li {
float: left;
width: $navbar-width/$items - 10px;

background-color:
  lighten($navbar-color, 20%);
&amp;:hover {
  background-color:
    lighten($navbar-color, 10%);
}

}
}

/ style.css /

#navbar {
width: 800px;
border-bottom: 2px solid #ce4dd6; }

#navbar li {
float: left;
width: 150px;
background-color: #e5a0e9; }

#navbar li:hover {
  background-color: #d976e0; }


插值

变量不仅可以可以被属性使用.你可以用 #{} 将他们插入到属性名或选择器.

<div class="compare">
  <pre class="sass">/* style.scss */

$side: top;
$radius: 10px;

.rounded- {
border-#{$side}-radius: $radius;
-moz-border-radius-#{$side}: $radius;
-webkit-border-#{$side}-radius: $radius;
}

/ style.css /

.rounded-top {
border-top-radius: 10px;
-moz-border-radius-top: 10px;
-webkit-border-top-radius: 10px; }

Mixins

Mixins 是Sass 的一个最强大的特征之一. 它允许你重复使用 – 属性和选择器 –从而不用重复的复制粘贴它们或者将它们放到一个没有语义的类里.

Mixins 使用“@mixin” 来定义,它可以接受成块的样式，然后可以在其他选择器使用 “@include”来引用它们.

<div class="compare">
  <pre class="sass">/* style.scss */

@mixin rounded-top {
$side: top;
$radius: 10px;

border-#{$side}-radius: $radius;
-moz-border-radius-#{$side}: $radius;
-webkit-border-#{$side}-radius: $radius;
}

#navbar li { @include rounded-top; }

#footer { @include rounded-top; }

/ style.css /

#navbar li {
border-top-radius: 10px;
-moz-border-radius-top: 10px;
-webkit-border-top-radius: 10px; }

#footer {
border-top-radius: 10px;
-moz-border-radius-top: 10px;
-webkit-border-top-radius: 10px; }

Arguments

当你给mixins 传递arguments的时候他的强大才展现出来. Arguments 是用括号包围的一组使用逗号分隔的变量列表.每次使用mixins的时候arguments 中的变量可以被赋予不用的值

mixin arguments 亦可以有默认值，就像你平常定义他们一样. 然后使用 mixin 的时候可以选择不传递值.

<div class="compare">
  <pre class="sass">/* style.scss */

@mixin rounded($side, $radius: 10px) {
border-#{$side}-radius: $radius;
-moz-border-radius-#{$side}: $radius;
-webkit-border-#{$side}-radius: $radius;
}

#navbar li { @include rounded(top); }

#footer { @include rounded(top, 5px); }

#sidebar { @include rounded(left, 8px); }

/ style.css /

#navbar li {
border-top-radius: 10px;
-moz-border-radius-top: 10px;
-webkit-border-top-radius: 10px; }

#footer {
border-top-radius: 5px;
-moz-border-radius-top: 5px;
-webkit-border-top-radius: 5px; }

#sidebar {
border-left-radius: 8px;
-moz-border-radius-left: 8px;
-webkit-border-left-radius: 8px; }

@import

样式表文件可能会变得很大. CSS 有一个 @import 允许你将样式分割到多个样式表文件, 但这样每一个样式表会占用一个HTTP 请求，这就是 Sass的 @import 指令为什么直接将将其他样式表融入到当前样式表的原因. 不仅如此,使用@import引入的文件中定义变量和 mixin 可以在当前文件中使用

Sass 对于要引入的文件有命名上的规范 (称为 “partials”):它们以下划线开始.让我们创建一个 partial _rounded.scss 来包含 rounded mixin.

为了都支持 .scss 和 .sass文件, Sass 允许被引入的文件没有后缀名.这意味着我们可以仅仅引入 "rounded", 而不是 "rounded.scss".

<div class="compare">
  <pre class="sass">/* _rounded.scss */

@mixin rounded($side, $radius: 10px) {
border-#{$side}-radius: $radius;
-moz-border-radius-#{$side}: $radius;
-webkit-border-#{$side}-radius: $radius;
}

/ style.css /

#navbar li {
border-top-radius: 10px;
-moz-border-radius-top: 10px;
-webkit-border-top-radius: 10px; }

#footer {
border-top-radius: 5px;
-moz-border-radius-top: 5px;
-webkit-border-top-radius: 5px; }

#sidebar {
border-left-radius: 8px;
-moz-border-radius-left: 8px;
-webkit-border-left-radius: 8px; }

/ style.scss /

@import “rounded”;

#navbar li { @include rounded(top); }

#footer { @include rounded(top, 5px); }

#sidebar { @include rounded(left, 8px); }

jsdoc 介绍及中文乱码问题

2011-03-03

jsdoc是用来批量提取js注释生成文档的工具，很早就听说php等有这些工具，今天查了一下js的，jsdoc还不错，但是使用中遇到了中文乱码问题，先介绍编码问题了。

中文乱码问题

如果你的jsdoc出现编码问题，中文是乱码的话可以使用-e参数来制定编码，当然编码要和你的js文件格式一致，如果你的js是utf-8格式，则使用：-e=utf-8。
e.g [code=’java’]java -jar jsrun.jar app/run.js -a -e=utf-8 -t=templates/jsdoc test/*.js [/code]
以上代码和app目录同级的的test目录下的所有js导出文档。如果不在同级目录则要使用绝对路径。
