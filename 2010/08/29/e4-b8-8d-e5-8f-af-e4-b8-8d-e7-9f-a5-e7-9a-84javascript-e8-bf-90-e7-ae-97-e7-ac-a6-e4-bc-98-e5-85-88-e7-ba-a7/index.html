不可不知的JavaScript运算符优先级

2010-08-29


### 相同优先级顺序执行

[code=”js”]
var a = 1 && 2 && 3 // 3，1 && 2 1为真返回2；2 || 3， 2为真返回3 ；
var b = 1 || 2 || 3 // 1 ，1 || 2 1为真返回1；1 || 3 ，1为真返回1；
[/code]

“&&” 寻找false。例如： a && b ,如果 a 为true，直接返回b，而不管b为true或者false 。

“||” 寻找true。 与&&同理，例如：a || b ,如果 a 为false，直接返回b，而不管b为true或者false 。


像 && || 连续使用，是这样的如：
return a && b || c ,根据a来判断返回值，a 是 false 则肯定返回 c；如果 b , c 都是 true ，那么我们就可以根据 a 来决定b 还是 c ，如果 a 是 false 则返回 c，如果a是true 则返回 b。
a || b && c
根据优先级相当于先算 b && c ,然后和a 相 或；如果a是true，则返回a，不论是b或c，如果a是false，则如果b是false，返回b，如果b是true，返回c；
相当于一下三种情况

 >>> 1 || b && c
1
>>> 0 || 0 && c
0
>>> 0 || 1 && 2
2


### && 大于 ||

[code=”js”]
var a = 3 && 0 || 2; //2
var b = 3 && 1 || 2; //1
var c = 0 && 1 || 2; //2
var d = 0 && 0 || 2; //2

var e = 3 || 0 && 2; // 3
var f = 3 || 1 && 4;// 3
var g = 0 || 0 && 1;// 0
var h= 0 || 2 && 3; // 3

var i= a && b || g && h //(a && b) || (g && h)
[/code]

|| 常用来设置默认值，例：

[code=”js”]
function foo(b){
var a = b || 1;
alert(a)
}

foo(2) //2
foo() //1
[/code]

### == 大于 &&

[code=”js”]
var a= 1 && 3 == 2 ;//false
var b= 0 && 3 == 3; // 0
[/code]

### || 大于 ? :

[code=”js”]
var a =1 && 2 ? 0 :1 // 0 相当于 (1 && 2) ? 0 :1
var b = 1 || 2 ? 0 :1 // 0 相当于 (1 || 2) ? 0 :1
[/code]

### ++大于 == 大于 && 大于 ||

[code=”js”]
var b=0;
var a = 2 == 1 && b++ == 1; //false
[/code]

jQuery之 nth 函数有：

[code=”js”]
cur.nodeType == 1 && ++num == result
[/code]

### have a exercise

jquery 之prop 函数有类似于：

[code=”js”]
a && b == c && d == e && f ? g:h
[/code]

相当于(a &&( b == c ) && (d == e) && f ) ? g:h

jQuery之 event 包裹函数fix有：

[code=”js”]
e.which = (event.button & 1 ? 1 : (event.button & 2 ?3 : (event.button & 4 ? 2 :0 )))
[/code]

此种类型的运算符连续使用在jQuery是家常菜。

### 来个总结

[code=”js”]运算符 描述
. [] () 字段访问、数组下标、函数调用以及表达式分组
++ – - ~ ! delete new typeof void 一元运算符、返回数据类型、对象创建、未定义值
/ % 乘法、除法、取模
+ - + 加法、减法、字符串连接
< < >> >>> 移位
< <= > >= instanceof 小于、小于等于、大于、大于等于、instanceof
== != === !== 等于、不等于、严格相等、非严格相等
& 按位与
^ 按位异或
| 按位或
&& 逻辑与
|| 逻辑或
?: 条件
= oP= 赋值、运算赋值
, 多重求值
[/code]

+ 和++ 的优先级看是前自加还是后自加。
