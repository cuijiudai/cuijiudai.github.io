“return false” 和”e.preventDefault”的区别

2010-08-28

Posted: 09 Jul 2010 02:16 AM PDT

这俩个我们经常见到，废话不多说，看例子

[code=’js’]
$(“a”).click(function() {
$(“body”).append($(this).attr(“href”));
return false;
}
[/code]

这个效果大家应该知道，那么他和下例有啥区别：

[code=’js’]
$(“a”).click(function(e) {
$(“body”).append($(this).attr(“href”));
e.preventDefault();
}
[/code]

思考一下，有啥区别那？？？

区别是这样的，return false也阻止了冒泡。

This demo.

所以可以这样认为：

[code=’js’]
function() {
return false;
}

// 等价于

function(e) {
e.preventDefault();
e.stopPropagation();
}
[/code]

当然这个问题没有这么简单，你最好看看这里 Event order的介绍。
